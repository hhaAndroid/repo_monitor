# Repo 监控助手 - 需求文档

## 1. 项目总体需求

给定某个 GitHub repo 地址，定时监控 main 分支合入 PR，将符合用户需求的 PR 记录到特定的文档中，方便跟踪 repo 进展。

## 2. 核心目标

- 每天 8:30 自动监控 GitHub repo，监控 main 分支合入 PR
- 基于预定义需求精确筛选，只关注重要特性
- 新建的 PR 追踪文档要格式排版好看清晰
- 结构清晰：每个仓库独立文档，方便查看和管理
- 支持恢复：记录文档中要记录当前监控到的最新 PR 是哪一条，下一次查看时候不要重复监控
- 如果某个 repo 是第一次监控，则第一次仅仅查看最新的 30 个合入 PR 即可
- PR 追踪文档里面记录的 PR 信息要包括跳转链接，否则不好直接找到对应 PR
- 每天定时追踪时候，如果有新的更新内容就发消息通知我，如果没有任何更新则直接发消息告诉我：今天 xxx 库没有更新

## 3. 监控仓库和对应需求

### 3.1 Verl

#### 3.1.1 地址

GitHub 地址：https://github.com/verl-project/verl

#### 3.1.2 监控需求

**必须关注：**
- 重点关注新特性，例如新增算法，新功能等
- 重点关注大重构，包括性能和功能等

**排除（不记录）：**
- Bug 修复 (fix:)
- 文档 (doc:)
- 杂项 (chore:, ci:)
- 测试 (test:)
